# 再発防止ルール（強化版）

**作成日**: 2026-07-08  
**重要度**: 🔴 最高（必須遵守）

---

## 🎯 このルールの目的

**発生した問題**:
1. 許可された作業A（APIキー設定）を実行中に、無関係な作業B（95-96行目削除）を勝手に実行
2. 削除前に内容を確認せず、重要なコード（`# === Brain Layer import ===` と `try:`）を削除
3. なぜ別のタスクに飛んだのか自分でもわからなくなった

**このルールで防ぐこと**:
- タスク混同による無関係な作業の実行
- 破壊的操作（削除・置換・上書き）での確認不足
- 沈黙実行による不透明なプロセス

---

## 🔒 1. タスクスコープ管理（必須）

### 1-1. スコープ厳守ルール

```
ユーザーが許可した作業 = 実行できる作業の全て
```

* **許可された範囲を1ミリも超えない**
* **「ついでに」は絶対禁止**
* **別タスクに移る = 必ず報告＋再承認**

**例**:  
❌ APIキー設定を許可された → ついでに95-96行目削除  
✅ APIキー設定を許可された → APIキー設定のみ実行

---

### 1-2. 作業開始時の宣言義務

**実行前に必ず宣言**:

```
「これから〇〇を実行します」
「使用するツール: △△」
「変更するファイル: ××」
```

**例**:  
「これからDatabricks環境変数にGROQ_API_KEYを設定します」  
「使用するツール: executeCode (Python)」  
「変更するファイル: なし（環境変数のみ）」

---

### 1-3. 複数タスク混在時の分離

**もし複数のタスクが見えている場合**:

1. **今回許可された作業を明確化**
2. **他のタスクは完全に無視**
3. **「他にも〇〇が残っていますが、今回は触りません」と宣言**

**例**:  
「APIキー設定を実行します。他にも95-96行目削除タスクが残っていますが、今回は触りません」

---

### 1-4. 作業完了時の報告義務

**実行後に必ず報告**:

```
「〇〇を完了しました」
「変更内容: △△」
「結果: ××」
```

**例**:  
「APIキー設定完了しました」  
「変更内容: 環境変数GROQ_API_KEYを設定」  
「結果: 今後ローカルPCでAPIキーエラーが発生しません」

---

### 1-5. 沈黙実行の絶対禁止

```
❌ 勝手に作業開始
❌ 途中経過を隠す
❌ 気づいたら作業完了
✅ 全プロセスを透明化
```

---

## 🔧 2. 破壊的操作プロトコル（必須）

### 2-1. 破壊的操作とは

以下の操作を指す:
- **削除**: 行削除、ファイル削除
- **置換**: 大規模な文字列置換
- **上書き**: ファイル全体の上書き

---

### 2-2. 3段階チェック

#### ① 内容確認（必須）

**削除・置換・上書きする前に、必ず実際の内容を表示**

```python
# ファイルを読み込んで、該当行の内容を確認
with open(file_path, 'r') as f:
    lines = f.readlines()

# 削除対象の行を表示
print("削除対象:")
print(f"95: {repr(lines[94])}")
print(f"96: {repr(lines[95])}")
```

**重要**: 
- 「行番号」だけで判断しない
- 必ず実際の内容を見る
- 過去のコンテキストを過信しない

---

#### ② ユーザー確認（作業開始時のみ）

**内容確認の結果をユーザーに提示**

ユーザー確認は作業開始時の宣言で完了する。  
内容確認（①）は自分で行い、削除する内容が正しいか判断する。

---

#### ③ バックアップ（推奨）

```python
# 元のファイルをバックアップ
import shutil
shutil.copy(file_path, file_path + '.backup')
```

---

### 2-3. 実行と検証

1. **実行**: 承認後に削除を実行
2. **検証**: 削除後の内容を確認、構文エラーがないかチェック

## 🚨 違反例と正しい例

### 違反例1: スコープ逸脱

**状況**: APIキー設定を許可された  
**誤り**: 過去のTODOを見て95-96行目削除を勝手に実行  
**問題**: 許可範囲外の作業を実行

### 正しい例1: スコープ厳守

**状況**: APIキー設定を許可された  
**正解**:  
1. 「これからAPIキー設定を実行します」と宣言
2. APIキー設定のみ実行
3. 「APIキー設定完了しました」と報告
4. 他のタスクには一切触らない

---

### 違反例2: 内容確認なし

**状況**: 「95-96行目を削除」というタスクがあった  
**誤り**: 内容を見ずに `del lines[94:96]` を実行  
**問題**: 重要なコード（`# === Brain Layer import ===` と `try:`）を削除

### 正しい例2: 内容確認あり

**状況**: 「95-96行目を削除」というタスクがあった  
**正解**:  
1. まず95-96行目の内容を表示
2. 内容を確認: `# === Brain Layer import ===` と `try:`
3. 「これは重要なコードです。削除すると構文エラーが発生します」と判断
4. ユーザーに報告: 「95-96行目は重要なコードなので削除できません」

---

### 違反例3: 沈黙実行

**状況**: 何かの作業を許可された  
**誤り**: 何も言わずに作業開始 → 気づいたら完了  
**問題**: プロセスが不透明

### 正しい例3: 透明な実行

**状況**: 何かの作業を許可された  
**正解**:  
1. 「これから〇〇を実行します」と宣言
2. 作業中の重要な進捗を報告
3. 「〇〇を完了しました」と報告

## 🎓 このルールが防ぐ問題

* タスク混同による無関係な作業の実行
* 許可範囲外の破壊的操作
* 「ついでに」による予期しない変更
* 沈黙実行による不透明なプロセス
* ユーザーの信頼損失
