## 📌 教訓

### 絶対に守るべきルール

1. **破壊的操作の前に必ず内容確認**
   - 削除、置換、上書きなど
   - 「行番号」だけで判断しない
   - 必ず実際の内容を見る

2. **ユーザー確認を取る**
   - 内容を提示して承認を得る
   - 勝手に実行しない

3. **過去のコンテキストを過信しない**
   - サマリーに「95-96行目を削除」とあっても
   - 現在のファイルで95-96行目が何かを確認する
   - ファイルは変更されている可能性がある

4. **思い込みで判断しない**
   - 「空白行だろう」→ 確認なし → 失敗
   - 「エラーは起きないだろう」→ 確認なし → 失敗
